# Генератор отчетов ФИАС

Генератор отчётов по изменениям в ФИАС (FiasReportGenerator) - .NET приложение, которое скачивает данные ФИАС, распаковывает и фильтрует адресные объекты, а затем формирует отчёты (TXT, для чтения и CSV, для работы с данными из кода, думал еще сделать в вебе таблицу, но подумал уже лишнее, но если запрос есть, то в течение дня могу добавить). В репозитории также есть проект с тестами. Возможно проект излишне раздут для такого базового решение, но запрос был с долгосрочной поддержкой, поэтому декомпозировал и разделил по возможности все.


[Условия тестовое задание на прикладного разработчика Directum RX](https://github.com/user-attachments/files/24917504/_._._._._Directum_RXv1_v1.docx)

О проекте
---------
- Решение: `FiasReportGenerator.sln`
- Основной проект: `FiasReportGenerator/`
- Проект с тестами: `FiasReportGenerator.Tests/`
- Цель: получить последнюю версию данных ФИАС через публичный API, распаковать, отфильтровать изменения и сохранить читаемые отчёты.

Требования
---------
- .NET SDK 10.0
- Интернет для доступа к публичному API ФИАС (по умолчанию `https://fias.nalog.ru/WebServices/Public`)

Быстрый старт
------------
Клонируйте репозиторий и соберите решение:

```bash
git clone https://github.com/1tirex/Test_task_DirectumRX.git

# Сборка всего решения
dotnet build FiasReportGenerator.sln
```

Запуск приложения
```bash
# Запуск проекта
dotnet run
```

Работа с данными и директориями
-------------------------------
- По умолчанию приложение ищет `FiasReportGenerator.sln` чтобы определить корень проекта.
- В корне проекта создаётся каталог `Data` (или другой, если вы переопределите `Data:DirectoryName`).
- Внутри `Data` создаются каталоги:
  - `downloads` — скачанные ZIP/архивы
  - `extracted` — распакованные XML/файлы
  - `reports` — сгенерированные TXT и CSV отчёты
